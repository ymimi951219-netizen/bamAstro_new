## 페이지 타입별 키워드 전략

### 홈페이지 (/)
**Target:** "[지역] 유흥" (모든 장르 포괄)
**Structure:**
- H1: "[지역] 유흥 정보 가이드"
- Meta: 모든 장르 언급
- Content: 6개 장르 brief intro + 링크

---

### 장르 가이드 페이지 ([region]-[genre]-guide.astro)
**Target:** "[지역] [장르] 완벽 가이드"
**Title:** "[지역] [장르] 완벽 가이드 | 가격·예약·팁·추천"
**Keyword Density:**
- Main keyword: 0.8-1.2% (약 8-12회 / 1000단어)
- LSI keywords: 자연스럽게 분산

**Structure:**
1. H1: "[지역] [장르] 완벽 가이드"
2. Intro (Featured Snippet 최적화) - 40-60 단어
3. H2: "1. [장르]란?"
4. H2: "2. 추천 업소"
5. H2: "3. 이용 시스템"
6. H2: "4. FAQ"
7. CTA: 전화 연결 버튼

---

### 비교 페이지 ([region]-[genre1]-vs-[genre2].astro)
**Target:** "[지역] [장르1] vs [장르2]"
**Title:** "[지역] [장르1] vs [장르2] | 완벽 비교"

**Structure:**
1. 비교 카드 (좌우)
2. 비교 테이블 (5-7개 항목)
3. FAQ (3-5개)
4. CTA

---

### 역명 페이지 ([region]/[station]/index.astro) - NEW
**Target:** "[역명] 유흥", "[역명] [장르]"
**Title:** "[역명] 유흥 가이드 | [지역]의 핫플레이스"

**Structure:**
1. H1: "[역명] 유흥 가이드"
2. Intro: 역 특징 설명
3. H2: "근처 [장르1] 추천"
4. H2: "근처 [장르2] 추천"
5. 지도 (optional)
6. 역 접근성 정보

---

### 블로그 포스트 (/blog/[slug].astro) - NEW
**Target:** 롱테일 + LSI 키워드
**Title:** "[지역] [상황] [장르] 완벽 가이드"

**예시 제목:**
- "[지역] [장르] 첫 방문 완벽 가이드"
- "[지역] [장르] 생일 파티 플랜 50선"
- "[지역] [장르] 회식 에티켓 가이드"
- "[지역] [장르] 초이스 팁 실패 안 하기"

**Target 단어수:** 2,000-3,500 (SEO 최적)

## 키워드 밀도 체크리스트

### 1,000단어 기준 (주 키워드 목표: 8-12회)

```
문서 작성 후 체크:

□ 주 키워드 ("[지역] [장르]"):
  - H1에 포함: 1회
  - H2에 포함: 1-2회
  - Intro: 1회
  - 정의 섹션: 1회
  - 시스템 설명: 2-3회
  - CTA: 1회
  - Meta Description: 1회
  합계: 8-12회 ✓

□ 보조 키워드 (3-5개):
  - 각각 2-5회 분산
  - 자연스러운 배치

□ LSI 키워드 (10-20개):
  - 관련 개념들 자연스럽게 포함
  - 과도하지 않음

□ 과도 최적화 체크:
  - 같은 단어 반복? ❌
  - 부자연스러운 어순? ❌
  - 가독성 저해? ❌
```

## 일간 체크리스트 (Daily Checklist)

### 콘텐츠 작성 시
```
□ 주 키워드 포함 (H1, Intro, 정의, 결론)
□ 보조 키워드 자연스럽게 분산
□ LSI 키워드 (동의어, 관련 개념) 포함
□ 가독성 체크 (문단 3-5줄, 줄바꿈 활용)
□ 문장 길이 체크 (한 문장 20-30글자 추천)
□ 리스트 형식 활용 (3개 이상 항목)
□ 이미지/테이블/그래프 포함
□ 메타 데이터 작성 (Title 55-65글자, Desc 150-160글자)
```

### 발행 전
```
□ 맞춤법 검사
□ 링크 확인 (모두 작동하는가?)
□ 이미지 파일명 확인 (한글 피함)
□ Schema.org 추가 (FAQSchema, HowToSchema 등)
□ Internal link 3-5개 추가
□ CTA 버튼 명확한가?
□ 전화번호 클릭 가능한가?
□ Mobile 미리보기 확인
```

## Naver vs Google 최적화 빠른 비교

| 요소 | Google | Naver |
|------|--------|-------|
| **Title 태그** | 60글자 권장 | 30글자 권장 |
| **Meta Desc** | 160글자 | 120글자 |
| **H 태그** | 계층 구조 중요 | 더 중요 |
| **Keyword 태그** | 무시함 | 가중치 있음 |
| **Schema.org** | 매우 중요 | 덜 중요 |
| **이미지 파일명** | 로마자 권장 | 한글 권장 |
| **블로그 백링크** | 보통 | 매우 강함 |
| **업데이트 빈도** | 보통 | 높을수록 좋음 |

## 자주 실수하는 부분 (Common Mistakes to Avoid)

```
❌ 하지 말 것:

1. 키워드 과도 반복
   "강남 가라오케는 강남 가라오케입니다..."

2. 메타 스터핑
   <meta name="keywords" content="키워드1, 키워드1, 키워드1...">

3. 부자연스러운 어순
   "가라오케의 강남은..."

4. 이미지 파일명 무시
   "image1.jpg" (✓ "gangnam-karaoke.jpg")

5. 한글 메타 무시
   <meta> 태그가 모두 영문

6. 내부 링크 앵커 약함
   <a href="/page">더보기</a> (✓ "강남 가라오케 가이드")

7. Naver 등록 안 함
   Google만 최적화

8. 콘텐츠 길이 너무 짧음
   500단어 미만은 랭킹 어려움

9. 원본 콘텐츠 없음
   복사 & 붙여넣기

10. 기술적 SEO 무시
    robots.txt, sitemap.xml, SSL 미설정
```
